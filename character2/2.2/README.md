# 机器学习评估方法可视化实验：留出法、交叉验证、自助法

这是一个面向初学者的 Streamlit 小项目，用来可视化演示《机器学习》（西瓜书）第 2 章 2.2 节“评估方法”。

你可以在页面中交互式选择：

- 数据集
- 模型
- 评估方法
- 数据规模、噪声、随机种子
- 留出比例、k 值、Bootstrap 次数等参数

然后实时观察：

- 数据是如何被划分成训练集、验证集、测试集的
- 留出法、k 折交叉验证、留一法、自助法分别怎样工作
- 每轮训练分数、验证分数、测试分数
- 为什么不同评估方法会得到不同的性能估计
- 为什么样本量少时单次留出法可能不稳定
- 为什么交叉验证通常比单次留出法更稳健

## 对应《西瓜书》2.2 的教学主线

### 什么是训练集、验证集、测试集

- 训练集：用于拟合模型参数
- 验证集：用于比较模型或调节超参数
- 测试集：用于最后一次、公正地估计泛化性能

### 为什么不能直接用测试集反复调参

如果你看一次测试结果就改一次模型，相当于模型间接“见过了”测试集。  
这会让测试分数变得偏乐观，失去作为最终泛化性能估计的意义。

### 为什么评估方法的目标是估计泛化性能

机器学习真正关心的，不是模型把当前训练数据背得多熟，而是它遇到“新样本”时表现如何。  
评估方法就是在有限样本下，尽量逼近这种未来表现。

### 为什么样本少时，单次划分不可靠

小样本下，一次偶然的切分就可能把容易样本或困难样本集中到验证集里，导致估计偏高或偏低。  
这正是留出法容易波动的原因。

### 为什么交叉验证通常更稳定

因为每个样本都轮流当过验证样本，结果不再只依赖单次切分，所以通常更稳健。

### 为什么 Bootstrap 的训练集会有重复样本

因为 Bootstrap 是“有放回采样”。  
样本每抽一次都会放回去，所以同一个样本可以被抽到多次，也会有一些样本一次都没被抽到。

### 为什么不同评估方法的结果不一定相同

因为它们使用的训练子集、验证子集、样本利用方式都不同。  
在噪声较大、样本较少、模型不稳定时，这种差异会更明显。

## 代码模块说明

- `app.py`：Streamlit 页面主程序，负责组织控件、图表、表格与教学说明
- `utils/data.py`：生成或加载数据集
- `utils/splitters.py`：手写实现留出法、重复留出法、k 折交叉验证、留一法、自助法
- `utils/models.py`：封装不同模型的创建
- `utils/metrics.py`：实现分类与回归指标
- `utils/evaluators.py`：组织评估流程、每轮训练与统计汇总
- `utils/plots.py`：绘制样本划分图、拟合曲线、得分分布图、方法对比图

## 说明

- 本项目有意把“评估逻辑”单独写在 `splitters.py` 和 `evaluators.py` 中，而不是直接调用 `cross_val_score` 之类黑盒接口。
- sklearn 只用于模型本体和少量数据集生成，这样更容易把《西瓜书》2.2 的核心思想看清楚。
